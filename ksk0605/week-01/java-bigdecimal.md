# 왜 BigDecimal 은 정확한 계산이 가능할까?

### 기본 실수 타입(float, double)은 이진 부동소수점(binary floating point) 

자바의 `float`, `double`은 IEEE 754표준에 따라 숫자를 2진수 기반 부동소수점으로 표현한다. 

따라서 아래와 같은 결과가 발생한다. 
```
System.out.println(0.1 + 0.2); // 0.30000000000000004
```

이유는 이진수의 특성상, 소수점을 가진 십진수 숫자들 중 상당수는 무한소수로 표현될 때가 많다. 
0.1 이라는 숫자 또한 0.00011001100110011... 의 이진수로 표현해야한다. 
따라서 컴퓨터는 근사치로 값을 저장하게 된다. 이로인해 숫자 계산에서 미세한 오차가 발생한다. 

### BigDecimal은 10진 기반 + 임의 정밀도로 동작

`BigDecimal`은 숫자를 정확한 10진수로 표현한다. 
즉, 0.1은 0.1 그대로 저장한다. 

BigDecimal은 내부적으로 두 값을 지닌다. 
* BigInteger intVal : 정수부를 그대로 저장 
* int scale : 소수점 자릿수

즉 0.1 이라는 숫자는 
`intVal = 1`, `scale = 1`로 저장된다.
